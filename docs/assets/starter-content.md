# Starter content

Phase P03 supplies five declarative, data-only packages under `content/`:

| Package | Purpose | Runtime assets |
| --- | --- | --- |
| `@infinite-snowball/starter-level` | Compact snowfield level contract | icon, arena GLB, layout GLB |
| `@infinite-snowball/starter-objects` | Collectible pebble and final goal stone | icon, two render GLBs, two collider GLBs |
| `@infinite-snowball/starter-character` | Decorative Pebble Friend | icon, model GLB, preview PNG |
| `@infinite-snowball/starter-campaign` | One-level original campaign | icon |
| `@infinite-snowball/starter-music` | Original `Snowdrift Signal` loop | icon, PCM WAV |

The packages contain no JavaScript, TypeScript, WebAssembly, HTML, CSS, executable hooks, or runtime fetch instructions. They validate against the P02 protocol and pass their exact manifest/package-reference hashes to P04, P05, P06, and P07.

## Source selection

The prototype model is the exact `Models/GLTF format/rock_smallA.glb` member from Kenney Nature Kit. The archive, member, preview, and embedded CC0 license text have immutable SHA-256 evidence in `tools/assets/sources/kenney-nature-kit/source-evidence.json`. Kenney is the authoritative publisher source; an asset aggregator is not accepted as sole evidence.

The retained source is a self-contained glTF 2.0 binary with no external URI, data URI, script, unreviewed required extension, or embedded texture. It is approximately 3 KiB and already far below the frozen triangle, material, and byte ceilings, so object and level outputs preserve its exact bytes rather than applying a lossy optimization with no measured benefit. The character output deterministically appends a real embedded `Idle` rotation animation with three keyframes because the P02 character contract requires at least one declared clip; tests and the headless smoke gate prove every declared clip exists in the generated GLB.

The icon and `Snowdrift Signal` are original deterministic outputs generated by `tools/assets/lib/asset-pipeline.mjs` and dedicated under CC0 1.0. The music is a four-second, 44.1 kHz, stereo PCM16 triangle-wave composition. It is not derived from or packaged with any commercial soundtrack.

## Rebuild and verify

```bash
pnpm assets:rebuild-starter
pnpm assets:check
```

`assets:rebuild-starter` deletes and recreates only the five generated starter-package directories, regenerates the one-to-one human and machine provenance ledgers, and writes `docs/assets/starter-content-budget.json` plus `docs/assets/p03-content-handoff.json`. Two clean rebuilds must produce byte-identical files.

The checked-in initial budget is 19 package files, about 0.74 MiB, seven GLBs, and 112 aggregate triangles. The machine-readable report is authoritative for exact current totals and hashes. `assets:headless-smoke` combines structural checks with a local Playwright/Three harness that loads every GLB through `GLTFLoader`, plays embedded clips, renders frames in WebGL, reads pixels, and fails on loader, console, request, or WebGL errors.

## Runtime handoff

P04 receives only manifest-derived package references, asset IDs and exact hashes, collider/render roles, bounds, and budgets through `docs/assets/p03-content-handoff.json`. `pnpm assets:handoff-check` rejects any drift between that checked-in handoff and the current manifests/runtime bytes. P04 owns loaders, R3F/Koota/Ecctrl integration, worker decisions, physics behavior, and telemetry. P03 does not add runtime code or claim that the retained rock is a complete arena, character rig, or production art set; it is a deliberately small cleared seam for the vertical slice.
