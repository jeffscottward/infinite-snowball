# Infinite Snowball Phase 05 — First Playable Vertical Slice

Phase ID: P05
Status: Planned
Owner role: Gameplay vertical-slice engineer
Depends on: P03, P04

## Goal and user value

Deliver the first complete 90-second Infinite Snowball run so players can roll, collect, grow, score, pause, restart, win or time out, and complete the starter arena offline after a persistent browser profile closes and reopens. P05 automation proves manifest installability plus SW/cache/profile persistence, not a real OS-installed-app lifecycle; P10 owns headed/manual install, full quit, and offline relaunch on shipping browsers/devices. This gate establishes deterministic gameplay reducers and keyboard/gamepad/touch parity before the catalog/store ecosystem begins.

## Prerequisites and dependencies

- P03 has delivered cleared starter assets, original or compatible music, provenance records, starter content manifests, and size-band budgets.
- P04 has delivered passing engine/input/gameplay and `@infinite-snowball/runtime-r3f` APIs, deterministic simulation, custom snowball control, normalized input/UI actions, camera intents, lifecycle/cleanup, and telemetry gates.
- No prior phase creates `apps/web/**`; P05 creates the minimal private package/shell and initial `/play` route under the workspace/check contract.
- P05 may add only existing P03/P04 workspace dependencies it directly imports, including the public `@infinite-snowball/runtime-r3f` composition surface. It must not predeclare P07 `@infinite-snowball/content-runtime` or P08 `@infinite-snowball/ui`, deep-import P04 packages, or import R3F/React Three Rapier/Three/Ecctrl directly around the adapter.
- P05 may begin only after P03 and P04 gates are available; P07 must not begin until the P05 stop condition is met.
- P06 may overlap P05 because it owns CLI/package workflow files and does not edit the playable route or web-shell integration.

## Stable inputs and contracts

- P04 `InputFrame`, fixed-step simulation, snowball controller, collection/growth reducers, camera intents, and telemetry helpers.
- P03 starter arena with three authored size bands, final goal object, cleared assets/music, license/provenance evidence, and budgeted GLB/audio outputs.
- Vertical-slice requirements: 90-second one-arena run, collection eligibility `ballRadius >= requiredRadius`, `newRadius = cbrt(oldRadius^3 + growthVolume)`, authored integer score, win/time-out, pause, restart on same level version/seed, and offline completion after closing/reopening a persistent browser profile. This is automated browser persistence evidence, not proof of OS installation.
- Both P05 ports implement a framework-neutral external-store seam: `getSnapshot(): Readonly<Snapshot>` and `subscribe(listener): () => void`, suitable for `useSyncExternalStore`. Repeated reads return the same object reference until a commit; each committed `stateVersion` creates one new immutable reference and emits exactly one notification. Snapshots include monotonic `stateVersion`, authoritative state, `busyCommand|null`, and stable error code/message. Every command returns `Promise<UiCommandResult>` where success carries the committed `stateVersion` and failure is a typed `busy|invalid-state|boot-failed|start-failed|restart-failed|save-failed|cleanup-failed` result; duplicate commands while busy return `busy` without side effects.
- `BootUiPort` exposes `booting|ready|offline-ready|fatal` plus `retryBoot()` and `startGame()`. Retry resolves only after the documented shell/runtime/starter readiness sequence reaches an authoritative state; Start rejects until ready/offline-ready and resolves only after the same seeded starter run is active.
- `GameplayUiPort` exposes immutable low-frequency run state plus `pauseRun(reason)`, `resumeRun()`, `restartSameSeed()`, and `quitRun()`. Pause resolves after ticks stop; resume only from a healthy paused run; restart resolves after the exact level version/seed is active; quit resolves successfully only after ticks stop, run subscriptions are disposed, and the last valid save commits. P08 may navigate only after that successful result and may wrap only these public methods.
- PWA and persistence baseline contract: the offline run uses only the P05-owned shell, manifest, service-worker registration, `apps/web/src/sw.ts`, `starter-precache`, versioned Dexie v1 `saves`/`settings` schema/adapter under `apps/web/src/db/**`, and the exact P03 starter artifacts required for the run; missing shell, starter responses, or starter persistence fail closed instead of falling through to network or P07 catalog/store behavior.
- Production CSP/WASM contract: the built app embeds a GitHub-Pages-compatible CSP meta baseline with `default-src 'self'`, `script-src 'self' 'wasm-unsafe-eval'`, and no script `'unsafe-eval'`/`'unsafe-inline'`; narrowly scoped self/blob/data style, image, font, media, connect, worker, and manifest directives; plus `object-src 'none'`, `base-uri 'self'`, and `form-action 'self'`. The P05 Chromium gate must use this embedded meta policy only and must not add or depend on a CSP response header. Only the pinned build-owned Rapier artifact may be emitted/fetched as WASM, it is same-origin and served as `application/wasm`, and P02 continues to reject community WASM. P11 may add stronger HTTP-only directives such as `frame-ancestors 'none'` only if the approved header and embedded meta intersect without weakening the policy.
- The CSP gate runs the real production build through a static server that returns actual MIME and no CSP header for the P05 Chromium proof, boots the Rapier-backed `/play` route, and observes a successful physics step. Its test harness loads the build-emitted inert same-origin external probe script (allowed by `'self'`) that installs a `securitypolicyviolation` listener, attempts `new Function`, and records both the thrown CSP error and violation event from the document realm; it separately inserts a nonce-less inline `<script>` and proves its marker never executes plus a violation fires. Direct Playwright/CDP `page.evaluate(() => new Function(...))` is forbidden as CSP evidence because automation worlds may bypass document policy.
- Base-neutral hosting contract: one production build uses Vite `base: "./"` and hash-route serialization. Serve checksum-identical bytes on separate origins for the root and `/infinite-snowball/` cases so a root-scoped worker cannot control the prefix case. Logical `/play` loads as `/#/play` or `/infinite-snowball/#/play`; chunks/icons/manifest/probe are relative; the P05 manifest uses stable `id: "./"`, `start_url: "./#/play"`, and `scope: "./"`; SW registration resolves `./sw.js` from `document.baseURI` with directory scope; SW URLs resolve against `self.registration.scope`; Rapier WASM stays module-relative. Assert prefix-correct `registration.scope` and `controller.scriptURL`. No host-specific rebuild, mutation, or SPA rewrite is allowed. P08 may later change only approved manifest fields and must rerun proof after build-affecting changes.
- Chromium installability never uses off-record fixtures. For each origin and run create a fresh/proven-empty normal `userDataDir` keyed to `{artifact hash, project, origin}`, launch `chromium.launchPersistentContext` without incognito, and verify the online document, SW script, required cache responses, CSP probe asset, and Rapier WASM response match the build/per-file hashes. After SW control send `Browser.getVersion`, `Page.enable`, and `Page.getInstallabilityErrors`; require empty errors. Fully close, then reuse that profile only for this run's offline relaunch; never reuse across projects, origins, artifact hashes, or reruns. Record logical key/options, Playwright package version, full Browser/installability responses, origin/hashes. `context.browser()` lookup, stale cache, synthetic prompt, unsupported response, and OS-install labels fail.
- Initial manifest floor on a secure-context-eligible origin is exact: `name: "Infinite Snowball"`, `short_name: "Snowball"`, stable `id: "./"`, valid `start_url: "./#/play"`, `scope: "./"`, `display: "standalone"`, `prefer_related_applications` absent, and real relative 192x192/512x512 PNG icons with exact `sizes`, `type: "image/png"`, `purpose: "any"`. SW fetch and icon bytes/MIME/dimensions/hashes must pass. P08 may change only start/description/theme/background; it preserves every other field, absence, icon, identity, and response.
- Unified package/boundary contract: `web-package-contract.test.ts` permanently asserts privacy, existing-only dependencies, required scripts, and phase-current exact allowlist; `runtime-boundary.test.ts` permanently rejects raw/deep 3D and non-browser imports. P07/P08 may update only explicit handed-off positive expectations after creating their packages and must rerun all permanent assertions.
- Router contract: `apps/web/src/app/router.tsx` uses the base-neutral hash router and registers only logical `/play` in P05; landing, splash, store, and settings wait for P08. Tests serve the same checksum-identical build on isolated root and project-prefix origins; route logic never hard-codes either prefix.
- Accessibility requirements: WCAG 2.2 AA baseline, semantic controls, focus trapping/restoration for pause, no color-only state, >=44x44 controls, reduced-motion-safe feedback, and 375 landscape-safe gameplay.

## Completion and stop condition

P05 completes only when all three primary gates and the supplemental `VG-05-CSP-WASM` stop gate pass, including a fresh `{artifact hash, project, origin}` normal profile per project/origin, verified online document/SW/cache/probe/WASM hashes, full Browser/empty installability CDP, same-run same-profile offline relaunch, and meta-only Chromium CSP proof. Cross-project/origin/hash/rerun reuse, stale/off-record/nonempty/simulated evidence, a P05 CSP response header, or `context.browser()` lookup blocks. P07 stays blocked; P10 later owns native lifecycle rows.
