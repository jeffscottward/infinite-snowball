# Infinite Snowball Architecture Decisions

Status: planned and reconciled. These decisions govern future P01-P11 implementation work only; this document contains no application implementation.

Canonical phase graph: `P01 -> P02 -> {P03 || P04}; P03 -> P06; {P03,P04} -> P05; {P05,P06} -> P07 -> P08 -> P09 -> P10 -> P11`.

## Accepted decisions

| ADR | Area | Decision | Rationale | Consequences and verification owner |
|---|---|---|---|---|
| ADR-001 | Product identity and UI system | Ship an original web-native rolling-collection arcade PWA with a playful winter toybox / rolling scrapbook identity. Use snow `#F8FAFC`, ink `#1E293B`, action `#2563EB`, accent `#F97316`, Baloo 2 display, Atkinson Hyperlegible body, 4/8 spacing, 8/12/16 radii, 150/220/300ms motion, and WCAG 2.2 AA gates. | Preserves the fun rolling premise while avoiding trade-dress and accessibility risk. | P08 owns splash, HUD, store, landing, responsive, reduced-motion, and accessibility evidence. Public copy says “original rolling-collection arcade game.” |
| ADR-002 | Workspace, packages, and dependency baseline | Use a private pnpm workspace. Bun may run compatible local TypeScript scripts. Do not add a task/build orchestrator until profiling or build complexity proves need. Public CLI target is unscoped `infinite-snowball`; internal domains use stable workspace packages under `packages/*` and remain private until npm scope ownership is verified. | Keeps build mechanics simple, reproducible, and reversible. | P01 resolves and freezes exact compatible pins: React/DOM 19.2.7, Three 0.185.1, R3F 9.6.1, Drei 10.7.7, React Three Rapier 2.2.0, Koota 0.6.6, Ecctrl 2.0.0, Dexie 4.4.4/dexie-react-hooks 4.4.0, Vite 8.1.4/vite-plugin-pwa 1.3.0/Workbox 7.4.1, Zod 4.4.3, glTF Transform 4.4.1/gltfjsx 6.5.3, Vitest 4.1.10, Playwright 1.61.1, TypeScript 7.0.2, pnpm 11.13.0. |
| ADR-003 | Runtime ownership and CSP | Engine uses Koota core tag/AoS traits and project-owned ordered systems; no JS `eval`/`Function`/`unsafe-eval`. `runtime-r3f` alone integrates the pinned Rapier WASM world and visible snapshot bridge. Production CSP permits only `'wasm-unsafe-eval'` for the pinned same-origin engine WASM served as `application/wasm`; community WASM remains protocol-forbidden. | Prevents JS dynamic-code, transform/step duplication, and frame-loop state churn while acknowledging Rapier's real WebAssembly instantiation requirement instead of claiming an impossible blanket CSP. | P04 proves no JS dynamic-code path and one runtime owner. P05 owns the supplemental meta-only Chromium proof with a build-emitted inert same-origin external probe asset path/hash and handoff hashes for artifact, policy, probe, and WASM. P10 consumes the latest post-P08 proof for cross-browser/manual Safari matrix rows, and P11 verifies live host delivery policy without weakening it. |
| ADR-004 | Simulation clock and rendering | Use fixed 60 Hz simulation with seeded PRNG, stable IDs, deterministic event sorting, at most four visible-frame catch-up ticks, hidden/tab-loss zero ticks, excess accumulation clamped with telemetry, and render interpolation between previous/current snapshots. | Gives repeatable gameplay tests without promising impossible cross-browser bitwise Rapier identity. | P04 owns reducer exact tests and pinned Rapier pose tolerance tests. P10 profiles input-to-simulation and frame-time budgets. |
| ADR-005 | Controller, Ecctrl, and growth | The snowball uses pure controller commands applied to a custom dynamic Rapier sphere with torque/impulses, braking, and speed/slope/air limits. Ecctrl 2.0.0 is optional for humanoid/on-foot/touch only through `EcctrlHandle.setMovement`, receives a complete input state including releases, and never drives deterministic ticks or the snowball. Growth is `newRadius = cbrt(oldRadius^3 + growthVolume)`; between ticks the one authoritative collider uses `setRadius`/`setMass`, followed by body mass-property recomputation. | A floating capsule/controller-owned frame loop is wrong for a rolling sphere; one collider plus aggregate volume is testable and avoids body churn. | P04 owns controller, installed-API contract tests, and Ecctrl boundary; P05 proves movement, collection, growth, score, camera, pause, restart, and win/time-out. |
| ADR-006 | Collection, streaming, and performance budgets | Collected bodies disappear by the next tick and become local-pose instanced visual attachments. Use deterministic ID-sorted spatial cells, at least three size bands, look-ahead, load/unload hysteresis, and Koota-owned residency. Do not unload player, attachments, or objective-critical entities. World rebasing is conditional on large-world profiles. | Avoids hundreds of live joints/bodies while allowing larger authored arenas. | Initial profile gates: low/mobile 30fps p95 <=33.3ms, <=80 draw calls, <=150k triangles, <=150 active bodies, heap <=250MB; mid/high gates scale as documented in the phase specs. P04/P10 own evidence. |
| ADR-007 | Worker policy | Start with main-thread R3F/Three and Rapier. Workers may handle bounded hashing/validation, archive inspection in CLI/CI, procedural/spatial planning, and supported decoders. Any worker ADR needs two traces proving >=20% frame-budget recovery and transfer overhead <25% of saved time. OffscreenCanvas additionally requires renderer dominance and loader/input/debug parity. | Avoids speculative complexity before a bottleneck exists. | P04 records traces before worker expansion; P10 rejects unproven worker migrations. |
| ADR-008 | Content protocol and browser no-code boundary | Define one closed-world Zod protocol with generated JSON Schema for six data-only kinds: level, character, object-pack, campaign, music, bundle. Browser imports only validators/types and never Node/npm/archive logic. Capabilities are empty in v1 and unknown fields are rejected. | One protocol shared by CLI, catalog CI, and browser prevents drift and arbitrary code execution. | P02 owns schema, threat fixtures, and strict rejection evidence. |
| ADR-009 | Package, catalog, and CLI model | CLI/CI resolves exact npm tarballs without lifecycle scripts. V1 transforms approved bytes into frozen `./catalog/`. Protocol uses literal `CatalogSnapshot.resourceBasePath`, dedicated canonical `CatalogResourcePath` for entry screenshots/icon/package record, and `resourcePath` (not absolute `url`) for immutable/package-asset rows; package-local `path` stays distinct. Browser uses the pure protocol resolver and never searches npm or executes content. | Static host-neutral bytes keep supply-chain review explicit and work at root/project prefixes. | P02 owns schema/resolver/negative fixtures; P06 emits canonical relative mappings; P07's separate Node-only tool materializes them and is excluded from browser graphs; P11 proves frozen bytes. |
| ADR-010 | Persistence, install transactions, and service worker | Dexie owns structured records; Cache Storage owns verified URL responses. P05 creates saves/settings v1; P07 atomically migrates catalog/install/SaveExport state. Install follows planned -> staging -> verifying -> committing -> installed or retained failed/canceled evidence. Page and worker inject bases derived from document base and registration scope into the same browser-safe resolver; only same-origin prefix-local `catalog/` results may be fetched/cached. | Structured recovery plus validated URL-addressable caches avoid host coupling and destructive failures. | P07 proves atomicity, refs, last-known-good recovery, literal-base/path rejection, dual-prefix containment, and no Node/materializer import in web/SW graphs. |
| ADR-011 | Threat model, provenance, and catalog curation | Reject traversal, absolute/drive/UNC/NUL/encoded path variants, symlinks, Unicode/case collisions, unknown files, JS/WASM/HTML/CSS, MIME mismatch, invalid hashes/semver/licenses, unsupported codecs, GLB external/data/network URIs, and bounded-extraction violations. Catalog PRs record npm provenance, integrity, per-file hashes, reviewer/date/evidence, budgets, licenses, and withdrawal/replacement. | Treats content packages as supply-chain inputs, not trusted app extensions. | P02 defines policy; P03 records asset provenance; P06/P07 enforce; P10 reviews. No v1 TUF/signature system beyond HTTPS, protected CI/review, exact locks, and per-file hashes. |
| ADR-012 | Assets, licensing, music, and local import | Prototype assets must be exact-artifact-cleared Quaternius, KayKit, or Kenney CC0. OS3A is discovery-only until original creator/source/license proof exists. Pelican is rejected as unverified. Runtime content is GLB-first through a reproducible pipeline. Music is original, commissioned, CC0, or CC-BY with attribution; NC/ND/ambiguous royalty-free grants are rejected. The linked Katamari soundtrack is never bundled, mirrored, streamed, suggested, cataloged, or auto-installed. | Clear provenance and original sound protect release viability. | P03 owns ledgers, transforms, budgets, and evidence. Local soundtrack import remains future local-only and never networked. |
| ADR-013 | Backend, offline, and deployment | v1 has no Supabase, backend account, cloud save, leaderboard, multiplayer, or D1. Save export/import is versioned and fail-closed. Publish one checksum-identical static artifact to Cloudflare Pages at `/` and GitHub project Pages at `/infinite-snowball/`; Vite base, hash routing, manifest id/start/scope, worker scope, embedded `./catalog/` paths, and release SHA/artifact/policy/probe/WASM hashes are fixed before P10 freeze and are never rewritten per host or at deployment. Mintlify source stays in repo. | Static-first avoids auth, abuse, privacy, cost, and host-specific builds until an online user story exists. | P07 owns SaveExport and scope-relative catalog behavior; P08 owns accessible UI and staged start route; P10 freezes dual-origin evidence and latest CSP proof; P11 deploys without rebuilding, proves Cloudflare header+meta intersection and GitHub Pages meta-only fallback, and runs Chromium host delivery plus selected clean native lifecycle instead of the full P10 matrix. |
| ADR-014 | Release, CI, and public packages | Required checks are `lockfile`, `types`, `unit`, `build`, `content-policy`, `license-provenance`, `package-pack`, `e2e-offline`, `dependency-review`, `codeql`, `secret-scan`; all eleven must reach terminal green for required repository surfaces. A no-check state is valid only for explicitly non-required surfaces. Pin GitHub Actions by full SHA, use least privilege, protected environments, CODEOWNERS for workflows/packages/catalog/licenses/deploy, immutable preview artifacts by commit SHA, and promote the identical artifact to production. P11 cutover mutations require the four external gates, clean HEAD/index at the P10-approved SHA, forbidden-tracked-path audit, terminal-green CI `secret-scan`, and final-SHA review evidence; normal reviewed pushes to the already-initialized public repo do not require Cloudflare/npm/Mintlify gates. Public packages use allowlists, trusted publishing/OIDC, and protected-CI dispatch/monitoring; local npm publish is forbidden. | Keeps release authority explicit and prevents a public-window or supply-chain accident. | P01 defines required checks; P10 approves the RC and matrix; P11 owns concrete `tools/release/**` cutover tools, preserve-public/convert-private mode, protected publisher dispatch, rollback, and live verification. |
| ADR-015 | Base-neutral dual-host artifact | Build once with Vite `base: "./"` and hash routing. Use relative manifest/icon/chunk URLs; stable manifest `id: "./"` and `scope: "./"`; page resources and frozen `./catalog/` paths resolved from `document.baseURI`; worker registration relative to that base; worker cache/fetch URLs from `self.registration.scope`; module-relative Rapier WASM; build-emitted inert CSP probe path retained with its hash. P05 starts at `./#/play`; P08 alone switches to `./#/start` after registering it. | Cloudflare root and GitHub project Pages can serve identical bytes without rewrite; isolated origins prevent a root worker from masking prefix defects. | P05/P07/P08 own staged tests and must rerun offline/CSP proof after build-affecting changes; P10 freezes checksum-identical isolated-origin automation plus manual real-install evidence; P11 deploys without host-specific rewrite and verifies live host policy/probe behavior. |


## Rejected alternatives and consequences

| Rejected alternative | Why rejected | Consequence of accepted path |
|---|---|---|
| Katamari naming, logo treatment, characters, UI trade dress, story, sounds, or soundtrack | Creates originality, licensing, and user-confusion risk. | All assets, copy, UI, and audio must be original or independently licensed; public descriptor stays generic. |
| Browser-side npm search, tarball extraction, package `import()`, `eval`, JS/WASM/HTML/CSS content, or lifecycle execution | Unsafe supply-chain model and impossible to secure as a casual in-browser content installer. | CLI/CI performs bounded extraction and validation; browser installs only curated immutable files. |
| Ecctrl as the snowball controller | Ecctrl is a floating-capsule character controller, not an authoritative rolling sphere controller. | Build a small custom Rapier sphere controller; Ecctrl remains a limited bridge for humanoid/on-foot/touch cases. |
| Variable-delta simulation or cross-browser bitwise physics promises | Causes nondeterministic gameplay and overpromises Rapier/browser consistency. | Use fixed ticks, sorted events, seeded reducers, and tolerance-based physics assertions. |
| Hundreds of collected objects as attached rigid bodies or joints | Increases physics cost and instability. | Disable collected bodies and attach visual instances with aggregate radius/mass changes. |
| OffscreenCanvas-first rendering | Adds loader, input, debug, fallback, and React/R3F synchronization complexity before profiling proves value. | Keep main-thread renderer; add workers only behind trace-based ADR gates. |
| OPFS or localStorage as primary persistence | OPFS is unnecessary for v1 immutable assets; localStorage is too small/synchronous for locks, saves, and catalogs. | Use Dexie plus Cache Storage; OPFS may be reconsidered only for later large mutable imports. |
| Supabase, D1, accounts, leaderboards, cloud saves, or multiplayer in v1 | Adds auth, abuse, privacy, retention, cost, and availability problems before core value is proven. | Static PWA plus local saves and export/import ships first. |
| Unfiltered npm search as the store catalog | Exposes users to unknown packages and review bypass. | Maintainer-curated static catalog with transparent PR evidence and exact versions. |
| Mandatory v1 TUF/signature layer | Adds ceremony before threat data justifies it. | Use HTTPS, protected CI/review, exact locks, and per-file hashes now; revisit if distribution risk changes. |
| Fake ratings/reviews, store badges, QR conversion, CRT/cyberpunk, or automatic Comic Neue body font | Conflicts with truthful launch status, original winter identity, and legibility baseline. | Landing uses real capture, Play/Browse Packs/Create a Pack CTAs, Atkinson Hyperlegible body, and accessible motion/contrast. |
| Copying dependency pins or architecture wholesale from Mario-Kart-3.js | Its pins are older and its architecture is only a reference, not a compatibility contract. | P01 verifies current pins from the reconciled snapshot and keeps package choices explicit. |

## External account and credential gates

Only these gates remain unresolved because they require account ownership, organization control, or credentials. They scope to P11 release/cutover mutations only: ordinary reviewed pushes to the already-initialized public repository do not require npm, Cloudflare, or Mintlify credentials. No phase may guess IDs, tokens, domains, or agent identifiers; no P11 public mutation may occur until all four gate groups are resolved, the exact P01 tracked/raw-output/secret audit is attached, clean HEAD/index at the P10-approved SHA is proven, the forbidden-tracked-path audit is clean, terminal-green CI `secret-scan` exists for that SHA, and final-SHA review evidence exists.

Historical repository fact for validators and handoffs: the public repository is <https://github.com/jeffscottward/infinite-snowball> (`PUBLIC`, default branch `main`), and only the existing Phase 01 commit was pushed when that public repository was initialized; later planning/implementation work remains local until explicitly reviewed and released.

| Gate group | Required external facts | First blocking phase | Stop condition |
|---|---|---|---|
| GitHub account/org/repo administration | Repository owner, visibility, branch protection authority, protected environments, CODEOWNERS enforcement, Actions permissions. | P11, after P10 release-candidate evidence. | Public repository and protected release workflows are not complete until admin access and protection evidence exist. |
| npm account/scope/publishing | Recheck unscoped `infinite-snowball`, organization scope ownership, 2FA, trusted-publishing/OIDC setup, package allowlists, dist-tag authority. | P11, with P01/P06/P09 dry-run preparation only. | No real publication before name/scope and trusted-publishing evidence pass; bad versions are deprecated and dist-tags restored, never overwritten. |
| Cloudflare account/project/DNS | Account/project credentials, Pages project, DNS/domain authority, deployment environment settings. | P11. | Cloudflare primary deployment is incomplete until the identical static artifact is live and rollback is verified. |
| Mintlify project/domain | Mintlify project access, domain credentials, production docs target. | P11. | Live documentation release is incomplete until the in-repo Mintlify source deploys and the public docs URL is verified. |
